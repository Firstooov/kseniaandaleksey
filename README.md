# Алексей ❤ Ксения — сайт-годовщина

Одностраничный сайт: тёмная тема, золотые акценты, анимации при прокрутке,
живой счётчик времени вместе, галерея с лайтбоксом, таймлайн истории,
письмо, фоновая музыка. Чистые HTML/CSS/JS — без фреймворков и без сборки,
открывается прямо в браузере.

## Структура проекта

```
kseniaandaleksey/
├── index.html              ← вся разметка страницы
├── css/style.css           ← все стили и анимации
├── js/main.js               ← вся логика (счётчик, лайтбокс, письмо, музыка…)
├── assets/
│   ├── photos/              ← сюда кладёте свои фото
│   ├── music/song.mp3       ← фоновая музыка (уже вставлена)
│   └── video/video.mp4      ← видео (по желанию, пока пусто)
└── README.md                ← этот файл
```

## 1. Как заменить фотографии

Фото сейчас показаны золотыми заглушками с подсказкой — это нормально,
так задумано, пока вы не положите файлы. Всё, что нужно — переименовать
свои фото и положить их в папку `assets/photos/` **с теми же именами**:

| Файл | Где используется |
|---|---|
| `hero.jpg` | Фон главного экрана |
| `gallery-1.jpg` … `gallery-8.jpg` | Фотогалерея (8 фото) |
| `timeline-1.jpg` … `timeline-5.jpg` | Фото рядом с этапами истории |
| `video-poster.jpg` | Превью-кадр видео-блока (если добавляете видео) |

Форматы `.jpg` / `.png` / `.webp` — любой подойдёт, просто поправьте
расширение в `index.html` в атрибуте `src`, если оно отличается
(например, было `hero.jpg`, а у вас `hero.png`).

Хотите больше или меньше фото в галерее/таймлайне — скопируйте один
блок `<figure class="gallery-item">…</figure>` (в `index.html`, секция
`<section class="gallery">`) или `<div class="timeline-item">…</div>`
(секция `<section class="timeline">`) и поменяйте номер файла, текст
и `data-side="left"/"right"` (для таймлайна — чередуйте).

Число в статистике «Общих фото» и «Ярких моментов» считается
автоматически по количеству карточек — вручную ничего не пересчитывать.

## 2. Как заменить текст

Весь текст лежит прямо в `index.html`, простым русским текстом —
искать точно по видимым фразам:

- **Имена и даты вверху** — секция `<section class="hero">`.
  Дата начала отношений задаётся в атрибуте `data-start="2025-07-24T00:00:00"`
  у блока `id="liveCounter"` — измените дату там, если нужно.
- **Даты и подписи в истории** — секция `<section class="timeline">`,
  каждая карточка `.timeline-card` содержит `<span class="timeline-date">`,
  `<h3>` (заголовок) и `<p>` (текст).
- **Письмо** — в самом низу файла, блок `<div class="letter-modal">`,
  внутри `<div class="letter-text">`. Текст сейчас написан от лица
  Алексея — замените на свои слова и подпись `<p class="letter-signature">`.
- **Цитата по центру страницы** — `<blockquote class="quote-text">`.
- **Подписи под фото в галерее** — `<figcaption>` внутри каждого фото.

Открывать можно любым текстовым редактором (Блокнот, VS Code, Notepad++).
Изменения видно сразу после сохранения файла и обновления страницы
в браузере (F5).

## 3. Музыка и видео

- Музыка уже лежит в `assets/music/song.mp3` — играет по клику на
  значок ноты ♪ в правом верхнем углу (браузеры не разрешают включать
  звук автоматически, поэтому нужен клик).
- Чтобы заменить трек — просто положите свой файл под тем же именем
  `assets/music/song.mp3` (либо своё имя — тогда поправьте путь в
  `index.html`, тег `<audio><source src="...">`).
- Видео пока не добавлено. Чтобы вставить своё — положите файл
  `video.mp4` в `assets/video/`, и кадр-превью `video-poster.jpg` в
  `assets/photos/`. Если удобнее вставить ссылку на YouTube — напишите,
  подскажу, как заменить блок на встроенный плеер.

## 4. Как посмотреть сайт локально

Просто откройте `index.html` двойным щелчком — сайт откроется в браузере.
(Часть современных браузеров может ограничивать автозапуск некоторых
скриптов при открытии файла напрямую без сервера — если что-то ведёт
себя странно, ниже есть команда для локального сервера.)

Если у вас установлен Python:
```
cd kseniaandaleksey
python -m http.server 8000
```
и открыть `http://localhost:8000` в браузере.

## 5. Как бесплатно опубликовать сайт через GitHub Pages

1. **Создайте аккаунт на GitHub** (если ещё нет) — [github.com](https://github.com).
2. **Создайте новый репозиторий**: кнопка «New repository», назовите,
   например, `kseniaandaleksey`, оставьте публичным (Public), кнопку
   «Add README» не нажимайте — он у вас уже есть.
3. **Загрузите файлы проекта**:
   - Самый простой способ — на странице пустого репозитория нажать
     «uploading an existing file», перетащить туда всю папку
     `kseniaandaleksey` (файлы `index.html`, папки `css`, `js`, `assets`)
     и нажать «Commit changes».
   - Либо через Git в терминале:
     ```
     cd kseniaandaleksey
     git init
     git add .
     git commit -m "Наш сайт-годовщина"
     git branch -M main
     git remote add origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git
     git push -u origin main
     ```
4. **Включите GitHub Pages**: в репозитории откройте вкладку
   **Settings → Pages** (в левом меню слева, раздел «Code and automation»).
   В блоке «Build and deployment» → «Source» выберите **Deploy from a
   branch**, ветку **main**, папку **/ (root)**, нажмите **Save**.
5. Через 1–2 минуты сайт будет доступен по адресу вида:
   ```
   https://ВАШ_ЛОГИН.github.io/НАЗВАНИЕ_РЕПОЗИТОРИЯ/
   ```
   Ссылку можно проверить там же, на вкладке Settings → Pages — она
   появится вверху блока с зелёной галочкой.

Готово — это постоянная бесплатная ссылка, можно отправлять кому угодно.
Каждый раз, когда вы обновите файлы в репозитории (загрузите новые фото
или поправите текст), сайт по той же ссылке обновится сам за
пару минут.

### Если фото не грузятся после публикации

GitHub чувствителен к регистру букв в именах файлов и путях
(`Hero.jpg` ≠ `hero.jpg`). Проверяйте, что имя файла в папке
`assets/photos/` в точности совпадает с тем, что написано в
`index.html`.
